## ⚠️ ส่วนที่ผมอนุมานเอง — ระวังการใช้

ส่วนเหล่านี้ **ไม่ได้อยู่ใน TOR ตรงๆ** หรือ **ผมเดาจาก context ภายนอก**

### ⚠️ 1. CII (Critical Information Infrastructure)
- **เปเปอร์อ้าง:** "ตร. ถูกจัดเป็น CII ตามมาตรฐานความมั่นคงปลอดภัยไซเบอร์แห่งชาติ"
- **TOR จริง:** หน้า 14 ข้อ 8.2.4 บอกแค่ "ความมั่นคงปลอดภัยโครงสร้างพื้นฐานสำคัญทางสารสนเทศของประเทศ" (15 คะแนน)
- **สถานะ:** การตีความเป็น "CII" ตามนิยาม สกมช. นั้นถูกต้องในวงการ แต่ TOR ไม่ได้ใช้คำว่า "CII" ตรงๆ
- **แนะนำ:** เปลี่ยนเป็น "ระบบต้องเป็นไปตามมาตรฐานความมั่นคงปลอดภัยโครงสร้างพื้นฐานสำคัญทางสารสนเทศของประเทศ (เทียบเท่ามาตรฐาน CII)"

### ⚠️ 2. 5 Roles (ADMIN / COMMANDER / STAFF / AUDITOR / VIEWER)
- **เปเปอร์อ้าง:** ระบบมี 5 บทบาท
- **TOR จริง:** หน้า 30 ข้อ 7.1.3 บอก "RBAC" + "Administrator" + กล่าวถึง "ผู้บังคับบัญชา" แต่ **ไม่ได้ระบุ 5 role ตรงๆ**
- **สถานะ:** ผมออกแบบเอง 5 role
- **แนะนำ:** ระบุว่า "ออกแบบเป็น 5 บทบาทตาม RBAC pattern (เป็นข้อเสนอของผู้พัฒนา)"

### ⚠️ 3. ROI Numbers (ทั้งหมด)
- **เปเปอร์อ้าง:**
  - "ลด 2,000 ชั่วโมง/ปี/ผู้บริหาร"
  - "ลดข้อผิดพลาด 30%"
  - "อัตรารายงาน 60% → 95%"
  - "ลดต้นทุนรายงาน 60%"
- **TOR จริง:** **ไม่มีตัวเลขเหล่านี้เลย**
- **สถานะ:** **ผมประมาณการเอง (made up)**
- **แนะนำ:** ลบออก หรือเปลี่ยนเป็น "ตัวเลขประเมินผลคาดการณ์ (ต้อง validate กับเจ้าของโครงการ)" — **ห้ามอ้างเป็นข้อเท็จจริง**

### ⚠️ 4. 5 Dashboard Types (ก้าวหน้า/พื้นที่เสี่ยง/ฉุกเฉิน/ทรัพยากร/ประสิทธิภาพ)
- **เปเปอร์อ้าง:** Dashboard 5 ประเภท
- **TOR จริง:** หน้า 32 ข้อ 8.10.10 พูดถึง Dashboard & Report แต่ **ไม่ได้ระบุชื่อ 5 ประเภทแบบที่ผมเขียน**
- **สถานะ:** มาจาก Claude.html (Claude เก่าใส่) — ยังไม่ได้ verify จริง
- **แนะนำ:** เปลี่ยนเป็น "Dashboard หลายประเภท (จัดประเภทตามผู้ใช้และข้อมูล)" หรือ verify ใน TOR หน้าที่ยังไม่อ่าน

### ⚠️ 5. External Integrations Beyond TOR
- **เปเปอร์อ้าง:** API กับ TDID/TDA, สศช., สำนักนายกฯ, กรมการปกครอง, กรมอุตุนิยมวิทยา, กรมการขนส่งทางบก
- **TOR จริง:** หน้า 30 ข้อ 6.4.2 ระบุชัด:
  - ✅ ศูนย์รับแจ้งเหตุ ๑๙๑
  - ✅ ระบบกล้องวงจรปิด (CCTV)
  - ✅ ระบบข่าวกรอง (อช.)
  - ❌ ส่วนที่เหลือ (TDID, สศช., สำนักนายกฯ, ฯลฯ) — **ผมเดาเอง**
- **แนะนำ:** ลบ integration ที่ไม่อยู่ใน TOR หรือระบุชัดว่า "ข้อเสนอเพิ่มเติมของผู้พัฒนา"

### ⚠️ 6. Storage SAN (ระบุแยกออกมา)
- **เปเปอร์อ้าง:** "Storage SAN" เป็นรายการแยกในฮาร์ดแวร์
- **TOR จริง:** Page 36-37 มี FC 16Gbps ใน DB Node spec แต่ **ไม่ได้แยก "Storage SAN" เป็นรายการ**
- **สถานะ:** อนุมานจาก FC interface
- **แนะนำ:** ไม่ใส่ "Storage SAN x1 ชุด" แยก — รวมใน DB Node spec

### ⚠️ 7. 9 Workflow States — Wording
- **เปเปอร์อ้าง:** ร่าง / เสนอ / อนุมัติ / เผยแพร่ / รับทราบ / ปฏิบัติ / ส่งผล / ตรวจ / ปิด
- **TOR จริง (หน้า 27 ข้อ 4.1):** "ร่าง เสนอ อนุมัติ เผยแพร่ การจ่ายคำสั่ง รับทราบ เริ่มปฏิบัติ ส่งผลหลักฐาน ปิดงาน/ประเมินผล"
- **สถานะ:** ใกล้เคียง — แต่ TOR มี "การจ่ายคำสั่ง" + "ส่งผลหลักฐาน" + "ปิดงาน/ประเมินผล" (ที่ผมยุบเป็น "ตรวจ + ปิด")
- **แนะนำ:** ปรับให้ตรง TOR หรือระบุว่ารวบ "ตรวจ + ปิด" เป็นทางเลือกการ implement

### ⚠️ 8. PoC 2 Doc Classification — Scoring Rubric Detail
- **เปเปอร์อ้าง:** "TOR ไม่ระบุ % accuracy ตรงๆ"
- **TOR จริง:** หน้า 51 มี rubric แต่ตอนผมอ่านอ่านไม่ชัด (OCR distort)
- **สถานะ:** ผมเองยังไม่ verify 100%
- **แนะนำ:** กลับไปอ่านหน้า 51 อีกครั้งเพื่อยืนยัน

### ⚠️ 9. Public Service Awards / WSIS Prizes
- **เปเปอร์อ้าง:** "ระบบ EOP อาจได้รับการพิจารณารางวัล"
- **TOR จริง:** หน้า 2 พูดถึงรางวัลพวกนี้ในความเป็นมา — แต่ **ไม่ได้บอกว่าโครงการนี้ "จะ" ได้รางวัล**
- **สถานะ:** อ้างถูกแต่ใช้ผิดบริบท — TOR แค่บอกแรงบันดาลใจ
- **แนะนำ:** เปลี่ยนจาก "อาจได้รับรางวัล" เป็น "สามารถยกระดับเทียบกับมาตรฐานที่ได้รับการยกย่อง" หรือลบออก

### ⚠️ 10. หมวด Dashboard "5 ประเภท"
- ซ้ำกับข้อ 4 — รวมแล้ว

### ⚠️ 11. ทุนจดทะเบียน 3 เท่าของวงเงิน (~276M)
- **เปเปอร์อ้าง (ในไฟล์ TOR_SUMMARY.md):** ทุนจดทะเบียน ≥ 3 เท่าของวงเงิน
- **TOR จริง:** หน้า 5 ข้อ 3.12 ระบุ "มีมูลค่าของกิจการของบริษัทไม่ต่ำกว่าสาม (๓) เท่าของวงเงิน"
- **สถานะ:** อ่านถูกแต่ต้อง verify ว่า "มูลค่าของกิจการ" = ทุนจดทะเบียนจริงไหม (อาจเป็น net worth/assets)
- **แนะนำ:** เปลี่ยนเป็น "มูลค่ากิจการ ≥ 3 เท่าของวงเงิน" ตามคำใน TOR — ไม่ใช่ "ทุนจดทะเบียน" ที่อาจมีความหมายเฉพาะ
