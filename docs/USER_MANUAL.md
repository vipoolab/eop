## 1. ภาพรวมของระบบ

EOP เป็น Web Application สำหรับสำนักงานตำรวจแห่งชาติที่:

- **บูรณาการ 7 ระบบหลัก** ในแอปเดียว — ไม่ต้องสลับไปมา
- **AI ทำงานจริง 4 ฟีเจอร์** — ร่างคำสั่ง · จำแนกเอกสาร · OCR · ค้นหาแบบเข้าใจความหมาย
- **Workflow คำสั่ง 9 สถานะ** ตามมาตรฐานราชการ
- **Audit Log** บันทึกทุกการกระทำ ตรวจสอบย้อนหลังได้
- **MFA + RBAC** ความปลอดภัยระดับ CII

### 🎯 ผู้ใช้กลุ่มเป้าหมาย

| กลุ่ม | บทบาทหลัก |
|---|---|
| ผู้บังคับบัญชา (Commander) | อนุมัติคำสั่ง · ดู Dashboard · ลายเซ็นอิเล็กทรอนิกส์ |
| เจ้าหน้าที่ (Staff) | ร่างคำสั่ง · สร้างรายงาน · ใช้ AI ช่วยเขียน |
| ผู้ตรวจสอบ (Auditor) | ตรวจผลการปฏิบัติ · ดู Audit Log |
| ผู้ดูแลระบบ (Admin) | จัดการผู้ใช้ · ตั้งค่าระบบ |
| ผู้ดูข้อมูล (Viewer) | ดู Dashboard อย่างเดียว |

## 2. การเริ่มต้นใช้งาน

### 2.1 เปิดเว็บ

URL: `https://<deployed-url>` หรือ `http://localhost:3000` (เครื่อง dev)

### 2.2 ล็อกอิน

ที่หน้าแรกจะ redirect ไป `/login`:

1. กรอก **อีเมล** (เช่น `commander@eop.test`)
2. กรอก **รหัสผ่าน** (เช่น `demo1234`)
3. กด **"เข้าสู่ระบบ"**

### 2.3 บัญชีทดสอบ

หน้า Login มีปุ่ม **"คลิกบัญชีเพื่อกรอกอัตโนมัติ"** ทางขวา — เลือกได้ 4 บัญชี:

| บัญชี | บทบาท | ใช้ได้ |
|---|---|---|
| `commander@eop.test` | COMMANDER | ทุกอย่าง · อนุมัติคำสั่ง |
| `staff@eop.test` | STAFF | ร่างคำสั่ง · ใช้ AI |
| `admin@eop.test` | ADMIN | จัดการผู้ใช้ |
| `auditor@eop.test` | AUDITOR | ดู audit log · ตรวจผล |
| `viewer@eop.test` | VIEWER | ดูเท่านั้น |

**รหัสผ่านทุกบัญชี:** `demo1234`

### 2.4 หน้าหลัก (Dashboard)

หลังล็อกอินสำเร็จ → ระบบจะพาคุณไปที่ `/dashboard` อัตโนมัติ

หน้า Dashboard แสดง:
- ทักทาย: "สวัสดี [ยศ] [ชื่อ]"
- **6 KPI Cards** — สถิติคำสั่งจริงจาก DB
- **กราฟแนวโน้ม** 12 เดือน
- **สถิติสถานะคำสั่ง** real-time
- **กิจกรรมล่าสุด** จาก audit log

### 5 บทบาท · ระดับสิทธิ์จากสูงสุดถึงต่ำสุด

#### 🔴 ADMIN (ผู้ดูแลระบบ)
- ✅ ทำได้ทุกอย่าง
- ✅ สร้าง · แก้ไข · ระงับ ผู้ใช้
- ✅ ตั้งค่าระบบ
- ✅ ดู Audit Log ทั้งหมด

#### 🔵 COMMANDER (ผู้บังคับบัญชา)
- ✅ อนุมัติคำสั่ง + e-Signature
- ✅ สร้างคำสั่งใหม่
- ✅ ใช้ AI ทุกฟีเจอร์
- ✅ ดูทุก dashboard
- ❌ ไม่สามารถจัดการผู้ใช้

#### 🟢 STAFF (เจ้าหน้าที่)
- ✅ ร่างและส่งคำสั่งเพื่อพิจารณา
- ✅ ใช้ AI ช่วยร่าง / จำแนกเอกสาร / OCR
- ✅ ดู Dashboard
- ❌ ไม่สามารถอนุมัติคำสั่ง

- ✅ ตรวจผลการปฏิบัติ (`audit` action)
- ✅ ดู Audit Log
- ✅ ดูข้อมูล `intel` (ข่าวกรอง)
- ❌ ไม่สามารถสร้างคำสั่ง

#### ⚪ VIEWER (ผู้ดูข้อมูล)
- ✅ ดูข้อมูลทุกหน้า
- ❌ ไม่สามารถสร้าง · แก้ไข · ใช้ AI
- ❌ ไม่เห็นข้อมูลข่าวกรอง

## 5. Workflow ตัวอย่าง — การสร้างคำสั่ง

### Step 1: สร้างคำสั่งใหม่

1. ไปที่ **Command Workflow** ใน sidebar (ระบบ 4)
2. กด **"+ สร้างคำสั่งใหม่"**
3. กรอกข้อมูล:
   - **หัวเรื่อง** — ชื่อคำสั่งโดยย่อ (≥ 5 ตัวอักษร)
   - **ผู้รับ** — หน่วยงาน/บุคคล (เช่น "ผบก. ทุกหน่วย")
   - **อ้างถึง** — หนังสือก่อนหน้า (ไม่บังคับ)
   - **วัตถุประสงค์** — เป้าหมาย
   - **รายละเอียดคำสั่ง** — เนื้อหา (≥ 20 ตัวอักษร)
   - **ลำดับความสำคัญ** — LOW / NORMAL / HIGH / URGENT / CRITICAL
   - **ภารกิจที่เกี่ยวข้อง** (ไม่บังคับ)
   - **เลือกหน่วยรับ** — ≥ 1 หน่วย
4. กด **"บันทึกร่าง"**

➡️ ระบบจะ:
- สร้าง docNo อัตโนมัติ (เช่น "ตร ๐๐๐๑.๖๙/๐๐๔๒")
- สถานะเริ่มต้น = **DRAFT (ร่าง)**
- บันทึก audit log

### Step 2: ส่งเพื่อพิจารณา

1. กลับไป **Command Workflow** → คลิกการ์ดที่เพิ่งสร้าง
2. ในหน้า detail → กด **"ส่งเพื่อพิจารณา"**
3. ใส่หมายเหตุ (ไม่บังคับ) → กดยืนยัน

➡️ สถานะ: DRAFT → **SUBMITTED**

### Step 3: ผู้บังคับบัญชาอนุมัติ

1. Login เป็น **COMMANDER**
2. ไปที่ Command Workflow → เห็นการ์ดในคอลัมน์ **"เสนอ"**
3. คลิก → กด **"อนุมัติ"**

➡️ สถานะ: SUBMITTED → **APPROVED** (signerId บันทึก)

### Step 4: เผยแพร่

1. ในหน้า detail → กด **"เผยแพร่"**

➡️ สถานะ: APPROVED → **PUBLISHED** (publishedAt บันทึก)
➡️ ระบบจะแจ้งเตือนหน่วยรับ (in-app · email · LINE)

### Step 5: หน่วยรับทราบ + ปฏิบัติ + ส่งผล

แต่ละหน่วยรับ acknowledge → ปฏิบัติ → ส่งผล:

- **ACKNOWLEDGED** — หน่วยรับทราบครบ
- **IN_PROGRESS** — เริ่มปฏิบัติ
- **REPORTED** — ส่งผลการปฏิบัติ

### Step 6: ตรวจสอบและปิดงาน

- **AUDITED** — ผู้ตรวจสอบยืนยัน
- **CLOSED** — ปิดงาน · ประเมินผล

### 🎯 รวมทั้งหมด 9 สถานะตาม TOR 4.1:
```
ร่าง → เสนอ → อนุมัติ → เผยแพร่ → รับทราบ
     → ปฏิบัติ → ส่งผล → ตรวจ → ปิด
```

## 6. ฟีเจอร์ AI 4 ตัว

### 6.1 AI ร่างหนังสือสั่งการ (PoC 1)

📍 เมนู: **AI ร่างหนังสือสั่งการ** (ระบบ 2)

**วิธีใช้:**

1. **เลือก preset (เร็วสุด):**
   - ⚠️ ก่อเหตุประท้วง
   - 🚨 ภัยพิบัติฉุกเฉิน
   - 👑 งานสำคัญพิเศษ
   - 📋 งานทั่วไป

2. **หรือกรอกเอง 5 ช่อง:**
   - หัวเรื่อง · วัตถุประสงค์ · หน่วยรับ · ระยะเวลา · ความสำคัญ
   - (เสริม) บริบทเพิ่มเติม

3. กด **"ให้ AI ร่างให้"** → รอ ~10 วินาที

4. **ผลลัพธ์:**
   - อ้างถึง · วัตถุประสงค์ · เนื้อหา (ภาษาราชการ)
   - แสดง model · เวลา · tokens ที่ใช้

5. **บันทึก:**
   - กด **"คัดลอก"** (clipboard)
   - หรือ **"บันทึกเป็นร่างคำสั่งใหม่"** → ไปหน้า Create Command พร้อม prefill

### 6.2 AI Document Classification (PoC 2)

📍 เมนู: **AI Document Classification** (ระบบ 6)

**วิธีใช้:**

1. **ลากไฟล์** มาวางที่ drop zone หรือคลิกเพื่อเลือก
2. **รองรับ 5 รูปแบบ:**
   - DOCX (Word)
   - XLSX (Excel)
   - PDF
   - JPG / PNG
   - TXT
3. กด **"จำแนกด้วย AI"** → รอ ~3-10 วินาที

4. **ผลลัพธ์:**
   - **หน่วยงาน** (1 ใน 6 หมวด) + confidence %
   - **เหตุผลของ AI**
   - **สรุปเนื้อหา**
   - 🎯 Accuracy เป้าหมาย ≥ 85% (ทดสอบจริง 100%)

### 6 หน่วยงาน:
- **ยศ.** — กองยุทธศาสตร์
- **ผบ.** — กองแผนงานอำนวยการ
- **มค.** — กองแผนงานความมั่นคง
- **มข.** — กองแผนงานกิจการพิเศษ
- **วจ.** — กองวิจัย
- **อจ.** — ฝ่ายอำนวยการ สยศ.ตร.

### 6.3 OCR ภาษาไทย (PoC 3)

📍 เมนู: **OCR Demo** (ระบบ 6)

**วิธีใช้:**

1. **อัปโหลดภาพ / PDF** (JPG · PNG · PDF — สูงสุด 10 MB)
2. กด **"สกัดข้อความ"** → รอ ~10-25 วินาที

3. **ผลลัพธ์:**
   - ภาพต้นฉบับ (ซ้าย) — PDF จะแสดงใน iframe
   - **ข้อความที่สกัดได้** (ขวา) — **แก้ไขได้**
   - **ความมั่นใจ · จำนวนบรรทัด · เวลา · model**
   - 🎯 CER (Character Error Rate) เป้าหมาย ≤ 10% (ทดสอบจริง 1.40%)

4. **คัดลอก** ไปใช้ต่อ หรือ **ย้อนกลับ** เป็นต้นฉบับ

### 6.4 Intelligent Search 4 โหมด

📍 เมนู: **Intelligent Search** (ระบบ 6)

#### โหมด 1: พื้นฐาน (Basic)
- ค้นหาคำในหัวเรื่อง · เลขที่ · ผู้รับ
- เร็วที่สุด (~200ms)
- **ใช้เมื่อ:** รู้คำที่ต้องการชัดเจน

#### โหมด 2: ขั้นสูง (Advanced)
- Filter: **สถานะ · ความสำคัญ · ช่วงวันที่**
- กรอกคำค้นได้/ไม่ก็ได้
- **ใช้เมื่อ:** อยากกรองตามเงื่อนไข

#### โหมด 3: Full-text
- ค้นในเนื้อหา · วัตถุประสงค์ · อ้างอิง · เอกสาร
- AND ของหลายคำ (ต้องมีทุกคำ)
- **ใช้เมื่อ:** ต้องการหาในเนื้อหาทั้งหมด

#### โหมด 4: AI Semantic 🤖 (เด่นสุด)
- **เข้าใจความหมาย** · synonyms · context
- ใช้ Claude AI วิเคราะห์ความเกี่ยวข้อง
- **แสดง relevance score (%) + เหตุผลของ AI**
- ช้ากว่าโหมดอื่น (~5 วินาที)
- **ใช้เมื่อ:** ใช้ภาษาธรรมชาติ

**ตัวอย่าง:** ค้นว่า "งานที่เกี่ยวกับการประเมินผล"
- AI เข้าใจว่า "ประเมินผล" ≈ "ตรวจสอบผล"
- พบ: ตรวจสอบผลปฏิบัติงาน...ยาเสพติด (85% relevance)
- พบ: รายงานผลการกวาดล้าง... (80%)

## 10. ความปลอดภัย + Audit Log

### 📍 เมนู: **Security & Audit Log** (ระบบ 7)

### 10.1 Audit Log ดูอะไรได้บ้าง

ทุกการกระทำในระบบ:
- `auth.login` / `auth.login.failed`
- `command.create` · `command.submit` · `command.approve`
- `ai.command.draft` · `ai.document.classify` · `ai.ocr`
- `search.basic` · `search.fulltext` · `search.semantic`
- `user.create` · `user.activate` · `user.deactivate`
- `report.export.pdf`
- `notification.command.escalated`

แต่ละ entry บันทึก:
- **ผู้กระทำ** (ยศ + ชื่อ + role)
- **เวลา** (timestamp)
- **เป้าหมาย** (เช่น `command:abc123`)
- **รายละเอียด** (JSON: docNo, tokens used, etc.)

### 10.2 ระบบรักษาความปลอดภัย

- 🔒 รหัสผ่าน **bcrypt one-way hash**
- 🔒 Session ใช้ **JWT** อายุ 8 ชั่วโมง
- 🔒 **MFA** support (Authenticator App)
- 🔒 **TLS 1.3** + **AES-256** at rest
- 🔒 **RBAC** 5 ระดับ
- 🔒 Audit log **เก็บ 7 ปี** ไม่สามารถลบ
- 🔒 Anti-bruteforce + Rate limiting

## 12. FAQ — คำถามที่พบบ่อย

### Q1: ลืมรหัสผ่าน?
**A:** ติดต่อ ADMIN เพื่อ reset · หรือใช้ feature "ลืมรหัสผ่าน" ในหน้า login (Phase 2)

### Q2: ทำไม AI ร่างคำสั่งช้า?
**A:** Claude API ใช้เวลา 8-12 วินาที สำหรับ draft · นี่คือเรื่องปกติ · ใช้ Haiku 4.5 ราคาประหยัด

### Q3: เปลี่ยน role ทำอย่างไร?
**A:** เฉพาะ ADMIN ทำได้ ใน `User & Access Management`

### Q4: ส่งคำสั่งให้กรรมการดูได้มั้ย?
**A:** ใช้ feature **Export PDF** ใน Compliance Reports — หรือ copy link หน้า command detail

### Q5: คำสั่งที่ส่งไปแล้ว แก้ไขได้มั้ย?
**A:** ทำได้แค่ในสถานะ **DRAFT** (ร่าง) เท่านั้น · หลังจาก SUBMITTED แล้วต้อง "ส่งกลับแก้ไข" (reject) ก่อน

### Q6: ข้อมูลปลอดภัยมั้ย?
**A:** ระบบสอดคล้อง CII (Critical Information Infrastructure) — รหัสผ่าน bcrypt · TLS 1.3 · Audit Log 7 ปี

### Q7: ใช้ AI กี่ครั้งก็ได้มั้ย?
**A:** ใช้ได้ตามความจำเป็น · ระบบมี budget control ที่ Admin ตั้งค่าได้

### Q8: ทำไมต้อง MFA?
**A:** เป็นมาตรฐาน TOR 7.1.2 + CII · ป้องกันการขโมยบัญชี · เปิดใช้ผ่าน Authenticator App

### Q9: Dashboard มี real-time data?
**A:** Yes — KPI Cards ดึงจาก DB · refresh เมื่อ navigate เข้ามาใหม่

### Q10: ระบบรองรับ Backup?
**A:** ใช่ — Database snapshot ทุก 4 ชม. + Immutable Backup (ransomware-resistant)

## 13. Troubleshooting

### ❌ ปัญหา: เข้าระบบไม่ได้
**ตรวจสอบ:**
1. อีเมล + รหัสผ่านถูกต้องมั้ย
2. บัญชียัง active อยู่มั้ย (ติดต่อ ADMIN)
3. Cookie ของเบราว์เซอร์เปิดอยู่มั้ย

### ❌ ปัญหา: AI ตอบ Error 529 "Overloaded"
**สาเหตุ:** Anthropic API ปริมาณการใช้สูง (ปัญหาฝั่ง provider)
**แก้:** รอ 1-2 นาที แล้วลองใหม่

### ❌ ปัญหา: Upload ไฟล์ไม่ได้
**ตรวจสอบ:**
- ขนาดไฟล์ ≤ 10 MB
- รูปแบบไฟล์ถูกต้อง (ดูจากแต่ละหน้า)

### ❌ ปัญหา: PDF Export ไม่มีอักษรไทย
**สาเหตุ:** ระบบใช้ Noto Sans Thai font ในตัว — ถ้าไฟล์ font หาย ติดต่อ Admin

### ❌ ปัญหา: หน้าจอแสดงผลผิดเพี้ยน
**แก้:**
1. Refresh เบราว์เซอร์ (Ctrl+F5)
2. Clear cache + cookies
3. ใช้ Chrome / Firefox / Safari version ล่าสุด

### ❌ ปัญหา: Mobile app เปิดไม่ได้
**ตรวจสอบ:**
- ติดตั้ง PWA แล้วใช่ไหม
- มี Internet
- เบราว์เซอร์มือถือ version ใหม่
